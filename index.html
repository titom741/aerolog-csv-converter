CSV to Flights.json, Pilots.json, and Aircrafts.json

This page reads a single CSV file, automatically detects known columns, validates the minimum required fields, and generates the three JSON files expected by the app.

Why this page exists

CSV exports often use labels that do not match the app. This tool helps normalize them before import so the generated JSON stays usable.

How to use it

Load a CSV, drag the expected labels onto the right columns, fix orange or red signals, then download the three JSON files.

Minimum fields

Flight: date, registration, pilot 1, and total aircraft or simulator time.
Aircraft: manufacturer and model.
Route: departure and arrival for non-simulator rows.

Output filenames are always set to Flights.json, Pilots.json, and Aircrafts.json.

Default settings

Used when the CSV does not provide some aircraft details.

Tip: keep Aircraft, AP, and an engine type that matches your fleet if the CSV does not include these columns.
